# ES2015 Module Basics - No `modules` or `define` keywords - Just `import` and `export` - No need to `use strict` - Multiple exports(!) Note: No more `modules` keyword. Instead, you simply `export`. No need to ‚Äòuse strict‚Äô -> all ES6 modules are in strict mode already. Like with other common Javascript module systems, everything declared in a module remain local to the module unless they‚Äôre explicitly exported.
Have to manually expose code. ```javascript // toppings.js const combo = ['pineapple', 'feta'], // üòã favoriteCombo = combo; // won't be exported export combo // pizza.js export class Pizza { constructor(toppings) { this.toppings = toppings; } } ``` Can export `function`, `class`, `var`, `const`, and `let`. Note: To expose code from inside a module we have to export it. We can name our exports with function, class, let, const, var.
```javascript // index.js import { combo } from './libs/toppings'; import { Pizza } from './libs/pizza'; let pizza = new Pizza(combo); ``` Note: Then we just have to import the exported code where we want it.
You can `export` a list‚Ä¶ ```javascript const plain = [], pepperoni = ['pepperoni'], combo1 = ['pineapple', 'feta'], combo2 = ['pepperoni', 'sausage', 'people']; export { plain, pepperoni, combo1, combo2 } ```
‚Ä¶and rename your `export`s. ```javascript const plain = [], pepperoni = ['pepperoni'], combo1 = ['pineapple', 'feta'], combo2 = ['pepperoni', 'sausage', 'people']; export { plain, pepperoni, combo1 as pineappleAndFeta, combo2 as meatLovers } ``` Note: Perhaps you have specialty combinations that share similar names, `combo1` and `combo2`. These names don‚Äôt tell you much about what kind of information they actually have‚Äìsimply that they‚Äôre combos, so when you export them you want them to have a more useful name.
Naturally, the same goes for `import`s. ```javascript // index.js import { plain as myPizza, meatLovers as ew } from './libs/toppings'; import { Pizza } from './libs/pizza'; let pizza1 = new Pizza(myPizza), pizza2 = new Pizza(ew); ```
You can also `import` as an object ```javascript // index.js import * as toppings from './libs/toppings'; let plain = new Pizza(toppings.plain); ```
But what about `default`?
CommonJS and AMD modules can only export once. Note: ES2015 modules see CommonJS and AMD modules as having one "default" export.
```javascript import _ from 'lodash'; // equivalent to var _ = require('lodash'); ```
```javascript // log.js export default console.log.bind(console) // equivalent to module.exports = console.log.bind(console); ``` Note: You can export defaults, too.
Re-exporting modules ```javascript export { default as plain } from "./pizzas/plain"; export { default as pepperoni } from "./pizzas/pepperoni"; export { default as pineappleAndFeta } from "./pizzas/pineappleAndFeta"; export { default as meatLovers } from "./pizzas/meatLovers"; ``` Note: You can re-export modules.
